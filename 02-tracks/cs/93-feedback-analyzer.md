# 93. 고객 피드백 분석기 (feedback-analyzer)
{: .fs-7 }

매장 후기·VOC·설문을 모아 **개선 인사이트**로
{: .fs-5 .fw-300 }

---

🎯 **언제 쓰나요?** 분기 매장 평가 / 신규 오픈 매장 운영 점검 / 본사 정책 건의
⏱️ **소요 시간** 60분
🧰 **준비물** 매장 후기 데이터 (네이버지도·구글·자체 설문, 익명화)
{: .fs-4 }

---

## 0. 에이전트 구성

```
🤖 데이터수집가     → 흩어진 후기 수집·정제
🤖 감성분석가       → 긍정·부정·중립 분류
🤖 주제분류전문가   → 핵심 토픽 추출
🤖 인사이트작성자   → 실행 가능한 액션 제시
```

---

## 1. 시나리오

> 강남점은 지난 분기 네이버지도 후기 60건, 자체 설문 40건을 받았습니다.
> 점장은 다음 분기 운영 개선안을 본사에 제출해야 합니다.
> "고객들이 정말 무엇을 좋아하고 무엇에 불만인지" 한눈에 보이는 보고서가 필요합니다.

## 2. 그대로 붙여넣기

### 🧩 단계 1 — 데이터 정제

```
다음 매장 후기 100건을 받았어. 정제해줘.

요구:
1) 중복 제거
2) 광고성·욕설 필터링
3) 별점·날짜별로 정리
4) 익명 ID 부여 (R001, R002 ...)

[후기 데이터]
[리뷰1] ★★★★★ 2026-04-01 "직원이 친절하고 설명을 잘해줘서..."
[리뷰2] ★★ 2026-04-03 "배송이 너무 늦었어요. 약속한 날짜를 안 지키네요..."
... (붙여넣기 또는 CSV 첨부)
```

### 🧩 단계 2 — 감성 분석

```
정제된 100건을 감성 분류해줘.

- 긍정 / 중립 / 부정 비율
- 별점과 텍스트 감성이 불일치하는 케이스 (★★★★★인데 부정 내용 등)
- 강한 부정 (★★ 이하) 모두 추출
```

### 🧩 단계 3 — 주제 분류

```
부정 후기에서 반복되는 주제 Top 5를 추출해줘.

각 주제별:
- 발생 빈도
- 대표 후기 2개 인용
- 점장 입장에서 즉시 통제 가능한지 vs 본사 정책 사안인지 구분
```

### 🧩 단계 4 — 인사이트 보고서

```
지금까지 분석을 점장 분기 보고서 형식으로 정리해줘.

목차:
1) 요약 (3줄)
2) 주요 지표 (긍정률, 평균 별점, 추천 의향)
3) 잘된 점 Top 3 (계속 유지)
4) 개선 필요 Top 3 (액션 아이템)
5) 본사 건의 사항 2개
6) 다음 분기 KPI 제안

분량: A4 2장
톤: 객관적 데이터 중심, 감정 배제
```

## 3. 결과 예시

### 감성 분포
```
긍정: 68% (★4-5)
중립: 14%
부정: 18% (★1-2)
별점-텍스트 불일치: 6건 (모두 ★5이지만 본문에 불만 포함)
```

### 부정 주제 Top 3
| 주제 | 빈도 | 통제 가능성 |
|---|---|---|
| 배송 일정 지연 | 12건 | 매장 통제 ← 즉시 개선 |
| 설치 기사 매너 | 7건 | 매장+본사 협력 |
| 비스포크 컬러 재고 | 5건 | 본사 정책 |

### 인사이트 예시
> **개선 액션 #1**: 배송 일정 변경 발생 시, 변경 즉시 고객에게 자동 알림 + 사과 메시지 발송 체계 구축. 예상 효과: 부정 후기 30% 감소.

## 4. 응용 팁

### 🎯 매장 간 비교
같은 프롬프트에 여러 지점 데이터를 넣으면 매장 간 벤치마킹 가능.

```
강남점 100건, 코엑스점 80건, 분당점 60건을 한꺼번에 분석해서
지점별 강점·약점을 비교 보고서로.
```

### 🎯 시계열 추적
매월 같은 형식으로 분석해 트렌드 추적. 6개월 누적 시 계절성 발견 가능.

### 🎯 본사 보고에 인용
보고서 결과를 본사 정례회의에 제출 시 **고객의 목소리(VOC)**가 객관 근거가 됩니다.

## 5. 현장 적용 체크리스트

- [ ] 분기 1회 정례 분석 일정 등록
- [ ] 분석 결과를 매장 직원 조회에 공유
- [ ] 개선 액션 3개를 다음 분기 KPI에 반영
- [ ] 본사에 정책 건의 사항 1건 제출
