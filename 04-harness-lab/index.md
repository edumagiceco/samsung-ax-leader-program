# Harness-Lab — 본인 하네스 만들기
{: .fs-8 }

5/27 발표의 핵심 도구. **명령 한 줄로 본인 부서용 하네스를 자동 생성**합니다.
{: .fs-5 .fw-300 }

---

## Harness-Lab이란?

> **하네스를 생성하는 하네스.**

5/20에 customer-support·real-estate-analyst를 직접 만들지는 않았습니다. 누군가가 미리 만들어 둔 것을 다운로드해서 썼죠. **Harness-Lab은 그 "누군가의 자리"를 여러분이 대신 해주는 도구입니다.**

명령 한 줄 — "가족 여행 준비 하네스 만들어줘" — 만 하면, 폴더·agents 5명·skills 3개·orchestrator까지 자동 생성됩니다.

5/27 본인 부서 MVP는 이 도구로 시작하는 게 가장 빠릅니다.

---

## 실행 전 설정 (필수)

다른 실습과 모드가 다릅니다. 반드시 다음 3가지를 변경하세요.

| 항목 | 값 |
|---|---|
| 모드 | **자동 모드** |
| 모델 | **Opus 4.7 1M** |
| 작업량 | **매우 높음** |

설정 위치: Claude Desktop Code 탭 하단 상태바 → 클릭하여 변경

---

## 실행 절차

### 1. harness-lab 폴더 준비
- `C:\claude-practice\max-samsung-harness-samples\harness-lab` 폴더를 복사
- `C:\claude-practice\harness-lab`에 붙여넣기

### 2. Claude Desktop에서 폴더 선택
- New session → 폴더 선택 → `C:\claude-practice\harness-lab` 선택
- 하단에 `harness-lab` 표시 확인

### 3. 시드 명령 입력

```
/harness-lab [본인이 만들고 싶은 하네스 한 줄 설명]
```

예시 (5/20에 강사가 시연한 것):
```
/harness-lab 가족 여행 준비 하네스 만들어줘
```

### 4. 생성 결과 확인
- `harness-lab` 폴더 안에 새 하위 폴더가 생성됩니다 (예: `trip-planning/`)
- 그 안에 `.claude/agents/`, `.claude/skills/`가 자동 작성됨

### 5. 생성된 하네스 실행

New session → 폴더 선택 → 새로 만들어진 폴더 선택 → 실행

```
/<생성된 오케스트레이터> [구체적 요청]
```

예시:
```
/trip-planning-orchestrator 제주도 2박3일 가족여행 계획 세워줘
```

## 본인 부서용 시드 명령 — 6팀 예시

본인 부서에서 시작점으로 쓸 만한 시드 명령 예시입니다. **본인이 매일 하는 일** 중 하나를 골라 변형하세요.

### AX G — AX 추진·전략
```
/harness-lab 분기별 AX 추진 보고서 작성 하네스 만들어줘.
부서별 진척 데이터를 받아 진단·우선순위·다음 분기 계획까지
한 번에 만드는 흐름.
```

### 영업팀 (8지점) — 매출·고객
```
/harness-lab 주간 매장 실적 액션 브리핑 하네스 만들어줘.
지역·채널 KPI 데이터를 받아 달성률·이상치·다음 주 액션을
지점장이 그대로 회의 자료로 쓸 수 있게.
```

### 재경팀 — 재무·세무·감사
```
/harness-lab 월간 결산 보고서 자동화 하네스 만들어줘.
부서별 비용·매출 데이터를 받아 분석·이상치 탐지·임원
회의용 1장 요약·세부 부록까지 한 번에.
```

### 인사·안전·지원 — HR·SOP
```
/harness-lab 신입사원 30/60/90일 온보딩 하네스 만들어줘.
직무별 학습 콘텐츠·멘토 매칭·진도 점검·평가 리포트까지
모든 단계를 자동화.
```

### 정보전략·정보보호 G — 보안·컴플라이언스
```
/harness-lab 분기 정보보호 컴플라이언스 점검 하네스 만들어줘.
점검 항목·증빙 수집·갭 분석·개선 권고·다음 분기 계획까지
규제 대응에 그대로 쓸 수 있는 보고서로.
```

### 경영진단 G — 진단·평가·KPI
```
/harness-lab 분기 경영진단 보고서 하네스 만들어줘.
주요 KPI·달성률·이상치·리스크 시나리오·다음 분기 권고
까지 한 번에.
```

{: .tip }
> 시드 명령이 구체적일수록 결과 품질이 올라갑니다. "내가 매일 하는 일 한 가지"를 1-2문장으로 묘사하는 게 핵심.

## 자주 발생하는 이슈

### Opus 1M·매우 높음 설정을 안 했더니 실패해요
- Harness-Lab은 토큰 사용량이 큽니다. **반드시 Opus 4.7 1M·매우 높음** 설정 필요

### 생성된 하네스가 너무 단순해요
- 시드 명령에 **"누가 / 어떤 데이터 / 무슨 출력 / 검증 누가"**를 명시하면 풍부해집니다

### 생성된 폴더 안에 agents가 5명보다 적어요
- 시드 명령에 명시적으로 "5명의 전문가 에이전트"를 요청하거나
- "이 일에 필요한 전문성 4가지와 reviewer 1명 구성"으로 입력
