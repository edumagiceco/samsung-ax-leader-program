# 바이브코딩 UI
{: .fs-8 }

웹의 기본 원리와 UI 용어를 이해한 뒤, 자연어 프롬프트로 간단한 홈페이지를 만들어 보는 기초 실습입니다.
{: .fs-5 .fw-300 }

---

## 이 페이지에서 하는 것

IT 비전문가가 코드를 먼저 외우는 대신, **웹 화면이 어떤 부품으로 구성되는지 이해하고 AI에게 원하는 UI를 정확히 지시하는 법**을 익힙니다.

- 웹페이지가 브라우저에서 보이는 기본 원리 이해
- HTML, CSS, JavaScript의 역할 구분
- 홈페이지 UI를 헤더, 히어로, 카드, CTA, 푸터로 나눠 보기
- Claude Code 같은 바이브코딩 도구에 UI 요구사항을 말하는 법 연습
- 간단한 원페이지 홈페이지를 만들고 수정 요청을 반복해 보기

{: .important }
> 바이브코딩은 "코딩을 전혀 모른 채 결과만 받는 것"이 아니라, **원하는 화면을 말하고, 결과를 보고, 다시 수정 지시를 내리는 반복 작업**입니다.

---

## 1. 웹의 기본 원리

웹페이지는 브라우저가 여러 파일을 읽어서 화면으로 보여주는 결과물입니다. 처음에는 아래 정도만 구분하면 충분합니다.

| 용어 | 쉬운 설명 | 홈페이지에서 보이는 예 |
|---|---|---|
| 브라우저 | 웹페이지를 보여주는 앱 | Chrome, Edge, Safari |
| URL | 웹페이지 주소 | `https://example.com` |
| HTML | 화면의 내용과 구조 | 제목, 문단, 이미지, 버튼 |
| CSS | 화면의 디자인 | 색상, 글자 크기, 여백, 배치 |
| JavaScript | 화면의 동작 | 메뉴 열기, 탭 전환, 버튼 클릭 반응 |
| 반응형 | 화면 크기에 맞게 바뀌는 설계 | PC에서는 3열, 모바일에서는 1열 |
| 호스팅 | 만든 페이지를 인터넷에 올려두는 것 | GitHub Pages, 사내 서버 등 |

### 한 문장으로 정리

```text
HTML은 내용, CSS는 디자인, JavaScript는 동작, 브라우저는 이 셋을 읽어 화면으로 보여주는 도구다.
```

---

## 2. UI를 부품으로 쪼개 보기

AI에게 "예쁜 홈페이지 만들어줘"라고만 말하면 결과가 흔들립니다. 먼저 화면을 부품으로 나눠 말해야 합니다.

| UI 부품 | 역할 | 프롬프트에서 쓰는 표현 |
|---|---|---|
| Header | 상단 메뉴와 로고 영역 | "상단에 로고와 메뉴 4개를 배치해줘" |
| Hero Section | 첫 화면의 큰 소개 영역 | "첫 화면에서 핵심 메시지와 버튼이 바로 보이게 해줘" |
| Section | 주제별 내용 영역 | "문제, 해결책, 기대효과 섹션으로 나눠줘" |
| Card | 정보를 묶어 보여주는 박스 | "주요 특징 3개를 카드로 보여줘" |
| CTA | 사용자의 행동을 유도하는 버튼/문구 | "문의하기 버튼을 눈에 띄게 만들어줘" |
| Footer | 하단 정보 영역 | "하단에 연락처와 저작권 문구를 넣어줘" |

{: .tip }
> 좋은 UI 프롬프트는 **대상, 목적, 섹션, 디자인 분위기, 제약조건**을 함께 말합니다.

---

## 3. 바이브코딩 UI 제작 흐름

실제 작업은 한 번에 끝내지 않고 아래 흐름을 반복합니다.

| 단계 | 할 일 | 예시 |
|---|---|---|
| 1 | 만들고 싶은 화면을 설명 | "팀 AX 과제 소개 홈페이지를 만들고 싶어" |
| 2 | AI가 파일을 만들게 요청 | `index.html`, `styles.css`, `app.js` 생성 |
| 3 | 브라우저에서 확인 | 글자가 잘 보이는지, 버튼이 명확한지 확인 |
| 4 | 마음에 안 드는 부분을 구체적으로 수정 요청 | "히어로 제목이 너무 길어. 더 짧고 강하게 바꿔줘" |
| 5 | PC와 모바일 화면을 함께 확인 | 카드가 모바일에서 1열로 보이는지 확인 |

## 5. 첫 생성 프롬프트

아래 프롬프트를 복사해 Claude Code에 실행합니다. 팀명과 과제명은 본인 상황에 맞게 바꿔도 됩니다.

```text
너는 IT 비전문가도 이해할 수 있는 웹 UI를 만드는 프론트엔드 디자이너이자 개발자다.

[실행 위치]
- Claude Desktop Code 탭에서 다음 폴더를 열고 실행한다: C:\claude-practice\vibecoding-ui
- 현재 폴더 안에 ax-project-homepage 폴더를 만들고 결과물을 생성한다.

[만들 결과물]
우리 팀의 AX 과제를 소개하는 간단한 원페이지 홈페이지를 만들어줘.

파일은 다음 3개로 나눠서 작성해줘.
- index.html
- styles.css
- app.js

[홈페이지 목적]
- 방문자가 우리 팀의 AX 과제가 무엇인지 빠르게 이해하게 한다.
- 현재 업무 문제, AI로 바꾸려는 방식, 기대효과를 한 화면 흐름으로 보여준다.
- 발표나 공유 시간에 브라우저로 열어 설명할 수 있게 한다.

[필요한 섹션]
1. Header: 왼쪽에는 "AX Project" 로고 텍스트, 오른쪽에는 문제, 해결방식, 기대효과, 실습 버튼 메뉴
2. Hero Section: 팀 AX 과제를 한 문장으로 소개하는 큰 제목, 짧은 설명, "과제 흐름 보기" 버튼
3. 문제 섹션: 현재 AS-IS 업무에서 반복되는 문제 3개를 카드로 표현
4. 해결방식 섹션: AI 하네스 또는 AI 협업으로 바뀌는 TO-BE 흐름을 3단계로 표현
5. 기대효과 섹션: 시간 절감, 품질 향상, 공유 속도 개선 등 기대효과 3개
6. 실습 섹션: "샘플 내용 채우기" 버튼과 결과 미리보기 영역
7. Footer: 비공식 실습용 프로토타입임을 표시

[상호작용]
- "샘플 내용 채우기" 버튼을 누르면 가상의 팀명, 과제명, 기대효과 문장이 미리보기 영역에 표시되게 해줘.
- 상단 메뉴를 누르면 해당 섹션으로 부드럽게 이동하게 해줘.

[디자인 요구사항]
- 업무 교육용 자료처럼 차분하고 읽기 쉽게 만들어줘.
- 너무 화려한 랜딩페이지처럼 만들지 말고, 발표 자료로 쓰기 좋은 신뢰감 있는 스타일로 만들어줘.
- 카드 안에 또 다른 카드를 중첩하지 말아줘.
- PC와 모바일에서 모두 자연스럽게 보여야 한다.
- 글자가 버튼이나 카드 밖으로 삐져나가지 않게 해줘.

[데이터 규칙]
- 실제 회사 내부 정보나 개인정보는 넣지 말고, 모두 가상의 더미 데이터로 작성해줘.

완료 후 다음을 알려줘.
- 생성한 파일 목록
- index.html을 여는 방법
- 구현한 상호작용 목록
- 초보자가 수정해볼 만한 지점 3개
```

## 6. 결과를 보고 수정 요청하기

첫 결과가 마음에 들지 않아도 정상입니다. 바이브코딩은 결과를 확인한 뒤, 원하는 방향으로 다시 말하는 과정이 핵심입니다.

### 디자인 수정

```text
전체적으로 더 전문적인 업무용 교육 페이지처럼 보이게 수정해줘.
색상은 너무 단조롭지 않게 하되, 본문 가독성을 우선해줘.
버튼, 카드, 섹션 간 여백이 균형 있게 보이도록 다듬어줘.
```

### 문구 수정

```text
히어로 섹션의 제목이 너무 설명적이야.
발표 첫 화면에 어울리도록 더 짧고 강한 문장으로 바꿔줘.
대신 보조 설명 문장에 구체적인 내용을 넣어줘.
```

### 모바일 수정

```text
모바일 화면에서 카드와 버튼이 좁아 보여.
모바일에서는 카드가 1열로 쌓이고, 버튼 텍스트가 줄바꿈되어도 자연스럽게 보이도록 CSS를 수정해줘.
```

### 코드 설명 요청

```text
내가 IT 비전문가라고 가정하고, 이 홈페이지가 HTML, CSS, JavaScript에서 각각 어떤 역할로 나뉘어 있는지 설명해줘.
코드는 바꾸지 말고 주요 구조만 쉽게 설명해줘.
```

## 8. UI 디자인 갤러리: 이름을 알고 지시하기

UI를 잘 만들려면 "예쁘게"보다 **무엇을 어떤 스타일로 바꿀지** 말해야 합니다. 아래 갤러리는 [Yumeta lab의 Cursor AI를 위한 UI 디자인 갤러리](https://yumeta.kr/web/cursor/)에 나온 레이아웃, 컴포넌트, 디자인 스타일 분류를 바탕으로 수업용 프롬프트로 재구성한 자료입니다.

{: .tip }
> 디자인 지시를 할 때는 "구조와 문구는 유지하고, 시각 스타일만 바꿔줘"라고 먼저 말하면 내용이 엉뚱하게 바뀌는 일을 줄일 수 있습니다.

### 디자인 변경 기본 공식

```text
현재 홈페이지의 섹션 구조와 문구는 유지해줘.
시각 스타일만 [디자인 스타일명] 느낌으로 바꿔줘.
특히 배경, 카드, 버튼, 폰트, 여백, hover 효과를 그 스타일에 맞게 조정해줘.
모바일에서도 글자와 버튼이 겹치지 않게 해줘.
```

---

### 8.1 레이아웃 이름 먼저 알기

웹페이지를 만들 때 자주 쓰는 큰 덩어리의 이름입니다. 이 이름을 알고 있으면 AI에게 화면 구조를 훨씬 정확히 지시할 수 있습니다.

| 이렇게 부름 | 무엇인가 | 이렇게 지시하면 됨 |
|---|---|---|
| Navbar / Navigation Bar | 상단 로고와 메뉴 영역 | "상단에 로고, 메뉴 4개, 오른쪽 CTA 버튼이 있는 네비게이션 바를 만들어줘." |
| Hero Section | 첫 화면의 가장 큰 소개 영역 | "첫 화면에 큰 제목, 짧은 설명, 주요 버튼이 있는 히어로 섹션을 배치해줘." |
| Features Section | 주요 기능·특징 소개 영역 | "핵심 특징 3개를 아이콘과 설명이 있는 카드로 보여줘." |
| About Section | 회사·서비스·프로젝트 소개 영역 | "프로젝트 배경과 목적을 소개하는 섹션을 본문 중간에 넣어줘." |
| Testimonials Section | 후기·추천사 영역 | "사용자 후기 3개를 카드 형태로 보여주는 섹션을 추가해줘." |
| Pricing Section | 요금제·패키지 비교 영역 | "3개의 요금제 카드를 나란히 배치하고 추천 요금제를 강조해줘." |
| Contact Section | 문의·연락 영역 | "이름, 이메일, 메시지를 입력하는 문의 섹션을 만들어줘." |
| Footer | 하단 정보 영역 | "푸터에 저작권, 연락처, 관련 링크를 정리해줘." |

### 8.2 컴포넌트 이름 익히기

컴포넌트는 UI 안에서 반복해서 쓰이는 작은 조작 단위입니다. "버튼 하나"보다 "모달을 여는 버튼"처럼 말하면 결과가 더 정확해집니다.

| 이렇게 부름 | 무엇인가 | 이렇게 지시하면 됨 |
|---|---|---|
| Card | 정보를 묶어 보여주는 박스 | "각 기대효과를 카드 컴포넌트로 분리해줘." |
| Button | 클릭 가능한 명령 요소 | "주요 버튼은 primary, 보조 버튼은 secondary 스타일로 구분해줘." |
| Modal | 기존 화면 위에 뜨는 팝업 창 | "자세히 보기 버튼을 누르면 설명 모달이 열리게 해줘." |
| Dialog | 확인·취소 같은 결정을 받는 대화상자 | "삭제 버튼을 누르면 확인 다이얼로그가 뜨게 해줘." |
| Accordion | 클릭하면 접히고 펼쳐지는 목록 | "FAQ는 아코디언 형태로 만들어줘." |
| Carousel / Slider | 여러 내용을 좌우로 넘기는 영역 | "고객 사례 3개를 캐러셀로 넘겨 볼 수 있게 해줘." |
| Tooltip | 마우스를 올렸을 때 나오는 짧은 설명 | "아이콘 버튼에는 hover 시 툴팁을 보여줘." |
| Input Field | 한 줄 입력창 | "이름과 이메일을 입력하는 input field를 배치해줘." |
| Textarea | 여러 줄 입력창 | "문의 내용을 적는 textarea를 추가해줘." |

### 8.3 디자인 스타일 갤러리

같은 홈페이지도 스타일 지시문에 따라 전혀 다른 인상을 줄 수 있습니다. 아래 문장을 그대로 복사해 "현재 화면을 ..." 뒤에 붙여보면 차이를 빠르게 느낄 수 있습니다.

| 디자인 이름 | 화면이 이렇게 바뀜 | AI에게 이렇게 말하기 |
|---|---|---|
| 글래스모피즘 | 카드와 버튼이 반투명 유리판처럼 보이고, 뒤 배경이 은은하게 비쳐 보임 | "현재 화면을 글래스모피즘 스타일로 바꿔줘. 배경은 은은한 그라데이션으로 만들고, 카드와 버튼은 반투명 배경, blur 효과, 얇은 밝은 테두리를 적용해줘. 단, 본문 글자는 충분히 진하게 유지해줘." |
| 뉴모피즘 | 배경과 카드가 같은 계열 색이고, 부드러운 안팎 그림자로 눌린 듯한 입체감이 생김 | "뉴모피즘 스타일로 바꿔줘. 연한 회색 배경 위에 카드와 버튼을 같은 색 계열로 만들고, 부드러운 밝은 그림자와 어두운 그림자를 함께 써서 입체감을 줘. 대비가 낮아지지 않게 텍스트는 진하게 해줘." |
| 스큐모피즘 | 버튼과 카드가 금속, 종이, 가죽처럼 실제 물건을 닮은 질감으로 보임 | "스큐모피즘 스타일로 바꿔줘. 버튼은 실제 누를 수 있는 물리 버튼처럼 그라데이션, 테두리, 안쪽 그림자를 넣고, 카드에는 종이나 패널 같은 질감을 살려줘. 너무 오래된 느낌이 나지 않게 정돈해줘." |
| 모노 디자인 | 흑백 중심으로 정리되어 강한 대비와 명확한 정보 구조가 생김 | "모노 디자인으로 바꿔줘. 색상은 흰색, 검정, 회색만 사용하고, 얇은 검정 테두리와 명확한 타이포그래피로 구성해줘. 중요한 버튼만 검정 배경으로 강조해줘." |
| 플랫 디자인 | 그림자와 질감이 줄고, 단순한 색면과 깔끔한 2D 버튼으로 정리됨 | "플랫 디자인으로 바꿔줘. 복잡한 그림자와 질감을 제거하고, 단색 배경, 명확한 버튼 색상, 단순한 카드 레이아웃으로 정돈해줘. 업무용 페이지처럼 읽기 쉽게 만들어줘." |
| 브루탈리즘 | 굵은 테두리, 강한 색, 거친 배치로 실험적이고 도발적인 느낌이 남 | "브루탈리즘 스타일로 바꿔줘. 굵은 검정 테두리, 강한 원색 포인트, 큼직한 타이포그래피, 직설적인 버튼을 사용해줘. 단, 교육용 페이지라서 본문 가독성은 유지해줘." |
| 키치 디자인 | 밝고 과장된 색, 장식적 요소, 팝아트 느낌으로 화면이 장난스럽게 바뀜 | "키치 디자인으로 바꿔줘. 밝은 핑크, 노랑, 보라 같은 팝한 색을 쓰고, 버튼과 제목을 과장된 장식 스타일로 만들어줘. 너무 산만하지 않게 핵심 CTA는 하나만 강하게 보여줘." |
| 맥시멀리즘 | 패턴, 장식, 강한 색과 대비가 풍부해져 화면이 꽉 찬 인상을 줌 | "맥시멀리즘 스타일로 바꿔줘. 장식적 배경 패턴, 강한 색 대비, 큰 제목, 풍부한 카드 장식을 사용해줘. 정보가 묻히지 않도록 섹션 간 계층은 분명하게 유지해줘." |
| 레트로퓨처리즘 | 네온, 어두운 배경, 격자, SF풍 폰트로 복고적 미래 느낌이 생김 | "레트로퓨처리즘 스타일로 바꿔줘. 어두운 배경, 네온 포인트 색, 얇은 격자 패턴, 미래적인 제목 스타일을 적용해줘. 버튼은 빛나는 테두리 느낌으로 만들어줘." |
| 바이오모픽 | 둥근 곡선, 유기적 형태, 자연색 계열로 부드럽고 생동감 있는 화면이 됨 | "바이오모픽 스타일로 바꿔줘. 자연에서 온 듯한 곡선 형태, 둥근 카드, 초록과 하늘색 계열의 부드러운 그라데이션을 사용해줘. 섹션 배치는 편안하게 흐르도록 만들어줘." |
| 멤피스 | 80년대 팝아트처럼 점, 선, 기하학 도형과 선명한 색이 섞임 | "멤피스 스타일로 바꿔줘. 점, 지그재그, 원, 삼각형 같은 기하학 패턴을 배경 장식으로 쓰고, 선명한 노랑, 핑크, 청록을 포인트로 사용해줘. 카드와 버튼은 굵은 테두리로 표현해줘." |
| 아르데코 | 금색 포인트, 대칭 구조, 기하학 패턴으로 고급스럽고 클래식한 느낌이 남 | "아르데코 스타일로 바꿔줘. 어두운 배경에 금색 포인트, 대칭적인 레이아웃, 기하학적 테두리와 패턴을 적용해줘. 제목은 고급스러운 느낌으로 크게 보여줘." |
| 미니멀리즘 | 장식이 거의 사라지고 넓은 여백, 얇은 선, 적은 색으로 차분하게 정리됨 | "미니멀리즘 스타일로 바꿔줘. 불필요한 장식을 줄이고, 흰 배경, 넓은 여백, 얇은 구분선, 작은 색 포인트만 사용해줘. 텍스트 계층과 CTA는 명확하게 유지해줘." |

{: .warning }
> 업무용 교육 페이지에서는 브루탈리즘, 키치, 맥시멀리즘처럼 강한 스타일을 그대로 쓰면 피로도가 높아질 수 있습니다. 실습에서는 차이를 이해하는 용도로 적용하고, 실제 발표용 화면은 플랫, 미니멀리즘, 모노, 절제된 글래스모피즘 정도가 무난합니다.

### 8.4 같은 화면을 다르게 바꾸는 연습

아래 순서로 같은 홈페이지를 세 번 바꿔보면 "명령이 화면을 어떻게 바꾸는지" 감이 생깁니다.

```text
1단계: 현재 홈페이지를 플랫 디자인으로 정돈해줘.
2단계: 같은 구조를 유지한 채 글래스모피즘 스타일로 바꿔줘.
3단계: 다시 업무 발표용으로 적합하게 미니멀리즘 스타일로 정리해줘.
```

바뀐 결과를 볼 때는 다음 네 가지를 비교합니다.

| 비교 항목 | 볼 것 |
|---|---|
| 배경 | 단색, 그라데이션, 패턴, 어두운 배경 중 무엇으로 바뀌었는가 |
| 카드 | 테두리, 그림자, 투명도, 둥근 정도가 어떻게 바뀌었는가 |
| 버튼 | 색상, 크기, hover 효과, 강조 순위가 분명한가 |
| 글자 | 제목과 본문의 크기 차이, 대비, 읽기 쉬움이 유지되는가 |

## 9. 수강생용 프롬프트 템플릿

다른 주제의 홈페이지를 만들 때는 아래 틀을 사용합니다.

```text
나는 [대상]을 위한 [종류] 홈페이지를 만들고 싶어.

목적:
- [방문자가 하길 원하는 행동]

필요한 섹션:
- 상단 메뉴
- 첫 화면 소개 영역
- 주요 특징 3개
- 상세 설명
- 문의 또는 신청 버튼
- 푸터

디자인 분위기:
- [깔끔한 / 전문적인 / 친근한 / 고급스러운 / 미니멀한]

조건:
- 모바일에서도 잘 보여야 함
- 초보자가 이해하기 쉬운 구조로 작성
- HTML, CSS, JavaScript를 분리해서 작성
- 실제 개인정보나 사내 정보는 사용하지 않음
```
