# 바이브코딩 UI
{: .fs-8 }

웹의 기본 원리와 UI 용어를 이해한 뒤, 자연어 프롬프트로 간단한 홈페이지를 만들어 보는 기초 실습입니다.
{: .fs-5 .fw-300 }

---

## 이 페이지에서 하는 것

IT 비전문가가 코드를 먼저 외우는 대신, **웹 화면이 어떤 부품으로 구성되는지 이해하고 AI에게 원하는 UI를 정확히 지시하는 법**을 익힙니다.

- 웹페이지가 브라우저에서 보이는 기본 원리 이해
- HTML, CSS, JavaScript의 역할 구분
- 홈페이지 UI를 헤더, 히어로, 카드, CTA, 푸터로 나눠 보기
- Claude Code 같은 바이브코딩 도구에 UI 요구사항을 말하는 법 연습
- 간단한 원페이지 홈페이지를 만들고 수정 요청을 반복해 보기

{: .important }
> 바이브코딩은 "코딩을 전혀 모른 채 결과만 받는 것"이 아니라, **원하는 화면을 말하고, 결과를 보고, 다시 수정 지시를 내리는 반복 작업**입니다.

---

## 1. 웹의 기본 원리

웹페이지는 브라우저가 여러 파일을 읽어서 화면으로 보여주는 결과물입니다. 처음에는 아래 정도만 구분하면 충분합니다.

| 용어 | 쉬운 설명 | 홈페이지에서 보이는 예 |
|---|---|---|
| 브라우저 | 웹페이지를 보여주는 앱 | Chrome, Edge, Safari |
| URL | 웹페이지 주소 | `https://example.com` |
| HTML | 화면의 내용과 구조 | 제목, 문단, 이미지, 버튼 |
| CSS | 화면의 디자인 | 색상, 글자 크기, 여백, 배치 |
| JavaScript | 화면의 동작 | 메뉴 열기, 탭 전환, 버튼 클릭 반응 |
| 반응형 | 화면 크기에 맞게 바뀌는 설계 | PC에서는 3열, 모바일에서는 1열 |
| 호스팅 | 만든 페이지를 인터넷에 올려두는 것 | GitHub Pages, 사내 서버 등 |

### 한 문장으로 정리

```text
HTML은 내용, CSS는 디자인, JavaScript는 동작, 브라우저는 이 셋을 읽어 화면으로 보여주는 도구다.
```

---

## 2. UI를 부품으로 쪼개 보기

AI에게 "예쁜 홈페이지 만들어줘"라고만 말하면 결과가 흔들립니다. 먼저 화면을 부품으로 나눠 말해야 합니다.

| UI 부품 | 역할 | 프롬프트에서 쓰는 표현 |
|---|---|---|
| Header | 상단 메뉴와 로고 영역 | "상단에 로고와 메뉴 4개를 배치해줘" |
| Hero Section | 첫 화면의 큰 소개 영역 | "첫 화면에서 핵심 메시지와 버튼이 바로 보이게 해줘" |
| Section | 주제별 내용 영역 | "문제, 해결책, 기대효과 섹션으로 나눠줘" |
| Card | 정보를 묶어 보여주는 박스 | "주요 특징 3개를 카드로 보여줘" |
| CTA | 사용자의 행동을 유도하는 버튼/문구 | "문의하기 버튼을 눈에 띄게 만들어줘" |
| Footer | 하단 정보 영역 | "하단에 연락처와 저작권 문구를 넣어줘" |

{: .tip }
> 좋은 UI 프롬프트는 **대상, 목적, 섹션, 디자인 분위기, 제약조건**을 함께 말합니다.

---

## 3. 바이브코딩 UI 제작 흐름

실제 작업은 한 번에 끝내지 않고 아래 흐름을 반복합니다.

| 단계 | 할 일 | 예시 |
|---|---|---|
| 1 | 만들고 싶은 화면을 설명 | "팀 AX 과제 소개 홈페이지를 만들고 싶어" |
| 2 | AI가 파일을 만들게 요청 | `index.html`, `styles.css`, `app.js` 생성 |
| 3 | 브라우저에서 확인 | 글자가 잘 보이는지, 버튼이 명확한지 확인 |
| 4 | 마음에 안 드는 부분을 구체적으로 수정 요청 | "히어로 제목이 너무 길어. 더 짧고 강하게 바꿔줘" |
| 5 | PC와 모바일 화면을 함께 확인 | 카드가 모바일에서 1열로 보이는지 확인 |

## 5. 첫 생성 프롬프트

아래 프롬프트를 복사해 Claude Code에 실행합니다. 팀명과 과제명은 본인 상황에 맞게 바꿔도 됩니다.

```text
너는 IT 비전문가도 이해할 수 있는 웹 UI를 만드는 프론트엔드 디자이너이자 개발자다.

[실행 위치]
- Claude Desktop Code 탭에서 다음 폴더를 열고 실행한다: C:\claude-practice\vibecoding-ui
- 현재 폴더 안에 ax-project-homepage 폴더를 만들고 결과물을 생성한다.

[만들 결과물]
우리 팀의 AX 과제를 소개하는 간단한 원페이지 홈페이지를 만들어줘.

파일은 다음 3개로 나눠서 작성해줘.
- index.html
- styles.css
- app.js

[홈페이지 목적]
- 방문자가 우리 팀의 AX 과제가 무엇인지 빠르게 이해하게 한다.
- 현재 업무 문제, AI로 바꾸려는 방식, 기대효과를 한 화면 흐름으로 보여준다.
- 발표나 공유 시간에 브라우저로 열어 설명할 수 있게 한다.

[필요한 섹션]
1. Header: 왼쪽에는 "AX Project" 로고 텍스트, 오른쪽에는 문제, 해결방식, 기대효과, 실습 버튼 메뉴
2. Hero Section: 팀 AX 과제를 한 문장으로 소개하는 큰 제목, 짧은 설명, "과제 흐름 보기" 버튼
3. 문제 섹션: 현재 AS-IS 업무에서 반복되는 문제 3개를 카드로 표현
4. 해결방식 섹션: AI 하네스 또는 AI 협업으로 바뀌는 TO-BE 흐름을 3단계로 표현
5. 기대효과 섹션: 시간 절감, 품질 향상, 공유 속도 개선 등 기대효과 3개
6. 실습 섹션: "샘플 내용 채우기" 버튼과 결과 미리보기 영역
7. Footer: 비공식 실습용 프로토타입임을 표시

[상호작용]
- "샘플 내용 채우기" 버튼을 누르면 가상의 팀명, 과제명, 기대효과 문장이 미리보기 영역에 표시되게 해줘.
- 상단 메뉴를 누르면 해당 섹션으로 부드럽게 이동하게 해줘.

[디자인 요구사항]
- 업무 교육용 자료처럼 차분하고 읽기 쉽게 만들어줘.
- 너무 화려한 랜딩페이지처럼 만들지 말고, 발표 자료로 쓰기 좋은 신뢰감 있는 스타일로 만들어줘.
- 카드 안에 또 다른 카드를 중첩하지 말아줘.
- PC와 모바일에서 모두 자연스럽게 보여야 한다.
- 글자가 버튼이나 카드 밖으로 삐져나가지 않게 해줘.

[데이터 규칙]
- 실제 회사 내부 정보나 개인정보는 넣지 말고, 모두 가상의 더미 데이터로 작성해줘.

완료 후 다음을 알려줘.
- 생성한 파일 목록
- index.html을 여는 방법
- 구현한 상호작용 목록
- 초보자가 수정해볼 만한 지점 3개
```

## 6. 결과를 보고 수정 요청하기

첫 결과가 마음에 들지 않아도 정상입니다. 바이브코딩은 결과를 확인한 뒤, 원하는 방향으로 다시 말하는 과정이 핵심입니다.

### 디자인 수정

```text
전체적으로 더 전문적인 업무용 교육 페이지처럼 보이게 수정해줘.
색상은 너무 단조롭지 않게 하되, 본문 가독성을 우선해줘.
버튼, 카드, 섹션 간 여백이 균형 있게 보이도록 다듬어줘.
```

### 문구 수정

```text
히어로 섹션의 제목이 너무 설명적이야.
발표 첫 화면에 어울리도록 더 짧고 강한 문장으로 바꿔줘.
대신 보조 설명 문장에 구체적인 내용을 넣어줘.
```

### 모바일 수정

```text
모바일 화면에서 카드와 버튼이 좁아 보여.
모바일에서는 카드가 1열로 쌓이고, 버튼 텍스트가 줄바꿈되어도 자연스럽게 보이도록 CSS를 수정해줘.
```

### 코드 설명 요청

```text
내가 IT 비전문가라고 가정하고, 이 홈페이지가 HTML, CSS, JavaScript에서 각각 어떤 역할로 나뉘어 있는지 설명해줘.
코드는 바꾸지 말고 주요 구조만 쉽게 설명해줘.
```

## 7. 실습 중 꼭 확인할 것

| 확인 항목 | 질문 |
|---|---|
| 목적 | 첫 화면만 봐도 무엇을 위한 홈페이지인지 알 수 있는가? |
| 구조 | Header, Hero, Section, Card, CTA, Footer가 구분되는가? |
| 가독성 | 제목, 본문, 버튼의 크기와 대비가 충분한가? |
| 모바일 | 화면을 좁혔을 때 카드와 버튼이 깨지지 않는가? |
| 상호작용 | 버튼 클릭, 메뉴 이동 같은 동작이 실제로 되는가? |
| 데이터 | 실제 사내 정보 대신 더미 데이터만 사용했는가? |

## 8. 수강생용 프롬프트 템플릿

다른 주제의 홈페이지를 만들 때는 아래 틀을 사용합니다.

```text
나는 [대상]을 위한 [종류] 홈페이지를 만들고 싶어.

목적:
- [방문자가 하길 원하는 행동]

필요한 섹션:
- 상단 메뉴
- 첫 화면 소개 영역
- 주요 특징 3개
- 상세 설명
- 문의 또는 신청 버튼
- 푸터

디자인 분위기:
- [깔끔한 / 전문적인 / 친근한 / 고급스러운 / 미니멀한]

조건:
- 모바일에서도 잘 보여야 함
- 초보자가 이해하기 쉬운 구조로 작성
- HTML, CSS, JavaScript를 분리해서 작성
- 실제 개인정보나 사내 정보는 사용하지 않음
```
