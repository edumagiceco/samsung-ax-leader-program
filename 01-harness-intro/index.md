# 하네스 10분 입문
{: .fs-8 }

비개발자도 이해할 수 있게, **매장 비유**로 설명합니다.
{: .fs-5 .fw-300 }

---

## 한 줄 정의

> **하네스(harness) = 여러 명의 AI 직원을 한 팀처럼 묶어, 정해진 절차로 일을 시키는 시스템**

## 매장 비유로 이해하기

매장에 손님이 가전 패키지 견적을 요청했다고 합시다.

| 매장 현실 | AI 하네스 |
|---|---|
| 영업사원: 고객 니즈 파악 | 🤖 **에이전트 1** — 고객 분석 |
| 견적 담당: 제품 조합·할인 계산 | 🤖 **에이전트 2** — 견적서 작성 |
| 상담실장: 최종 검토·승인 | 🤖 **에이전트 3** — 품질 검증 |
| 점장: 전체 흐름 지휘 | 🎯 **오케스트레이터** — 팀 조율 |

→ **AI 한 명에게 다 시키는 것**이 아니라, **여러 명이 역할 분담**해서 더 정확하고 일관된 결과를 냅니다.

## 왜 한 명이 아니라 팀인가?

| 단일 AI | 하네스 팀 |
|---|---|
| 한 번에 모든 걸 시키면 누락·실수 | 단계별 검증 |
| 결과가 매번 들쭉날쭉 | 표준 절차로 일관성 |
| 매번 처음부터 설명해야 함 | 매장용으로 미리 세팅 |
| 못 하는 일이 많음 | 전문 역할 조합으로 확장 |

## 워크샵에서 만져볼 도구

- **Claude Code**: 노트북에 설치하는 AI 도구. 하네스를 만들고 실행
- **에이전트(Agents)**: 특정 역할만 하는 AI 직원
- **스킬(Skills)**: 자주 쓰는 작업 절차를 미리 세팅
- **오케스트레이터(Orchestrator)**: 팀을 지휘하는 흐름

## 우리 매장에 어떻게 도움이 되나요?

- ⏱️ **반복 업무 자동화**: 견적서·응대 매뉴얼·보고서 작성 시간 70% 단축
- 📋 **표준화**: 누가 응대해도 똑같은 품질의 결과
- 🆕 **신입 빠른 적응**: 베테랑의 노하우를 AI 팀에 박제
- 📊 **데이터 활용**: 후기·VOC 분석에서 인사이트

## 5분 미니 실습 (Day 1 오전)

전체 참석자가 동시에 진행합니다.

```
1. 터미널 열기
2. claude 명령어 실행
3. "안녕, 가전 매장 직원이야. 30대 신혼부부에게
   추천할 냉장고 3가지를 골라줘"
4. 답변 받고 → 후속 질문 던지기
```

이 작은 대화 한 번이 하네스의 출발점입니다. 직군별 트랙에서는 이걸 **여러 에이전트 팀**으로 확장합니다.
