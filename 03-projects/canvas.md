## 캔버스 양식

다음 양식을 그대로 종이에 출력하거나 노션·구글 닥스에 옮겨 작성하세요.

```
┌─────────────────────────────────────────────────────────┐
│ 프로젝트명:                                              │
│ 팀명·팀원:                                               │
│ 소속 트랙: 영업 / CS / 마케팅 / 운영 / 보고              │
├──────────────────────┬──────────────────────────────────┤
│ 1) 문제 (Problem)    │ 2) 대상 (Who)                    │
│ - 매장의 어떤 일이   │ - 누가 사용하나                  │
│   불편한가?          │ - 어떤 직무 페르소나              │
│                      │                                  │
├──────────────────────┼──────────────────────────────────┤
│ 3) 산출물 (What)     │ 4) 사용 시나리오 (How)           │
│ - 무엇을 만드나      │ - 언제 쓰나                      │
│ - 형식 (스킬·문서·  │ - 어떤 입력 → 어떤 출력          │
│   에이전트 팀)       │                                  │
├──────────────────────┴──────────────────────────────────┤
│ 5) 활용할 사례 (어떤 하네스를 조합하나)                  │
│ - 메인:                                                  │
│ - 보조:                                                  │
├──────────────────────────────────────────────────────────┤
│ 6) 성공 기준 (Done)                                      │
│ - 정량:  (예: 1시간 걸리던 일을 10분으로)               │
│ - 정성:  (예: 신입도 따라할 수 있음)                    │
├──────────────────────────────────────────────────────────┤
│ 7) 리스크·제약                                          │
│ - 기술:                                                  │
│ - 데이터:                                                │
│ - 시간:                                                  │
├──────────────────────────────────────────────────────────┤
│ 8) 5분 발표 핵심 한 줄                                  │
│ - "우리는 ___을 위해 ___을 만들었고, ___ 효과가 있다"   │
└──────────────────────────────────────────────────────────┘
```

## 작성 예시 — "신혼 패키지 견적 자동화"

| 항목 | 내용 |
|---|---|
| 프로젝트명 | 30초 신혼 패키지 견적기 |
| 팀 | 강남점 영업팀 (4명) |
| 1) 문제 | 신혼 패키지 견적 만드는 데 30분 걸림 |
| 2) 대상 | 매장 영업사원 (특히 신입) |
| 3) 산출물 | 페르소나 입력 → 견적·스크립트·팔로업 자동 생성 스킬 |
| 4) 시나리오 | 고객 상담 5분 후 즉석 견적 출력 |
| 5) 활용 사례 | 메인: 48 sales-enablement, 보조: 50 pricing-strategy |
| 6) 성공 기준 | 30분 → 5분, 신입도 사용 가능 |
| 7) 리스크 | 가격 정책 변경 시 갱신 필요 |
| 8) 한 줄 | "30대 신혼부부 견적을 5분 안에, 신입도 만든다" |

## 멘토 코칭 라운드

09:30~10:15 동안 멘토가 1:N으로 각 팀을 돌며 다음을 점검합니다.

| 체크 | 질문 |
|---|---|
| 문제 명확성 | "이 문제 진짜 있어요?" 매장에 실제로 있는 일인지 |
| 산출물 구체성 | "어떤 형태로 누가 받나요?" 명확하게 |
| 5분 발표 가능 | "한 줄 발표 문장 지금 말해보세요" |
| 가능 범위 | "오늘 안에 끝낼 수 있나요?" 욕심 조절 |

캔버스가 통과되면 개발 단계로 진행합니다.
