# 5분 발표 템플릿
{: .fs-7 }

Day 2 마지막 발표를 위한 슬라이드 골격 + 진행 가이드
{: .fs-5 .fw-300 }

---

## 5분 = 슬라이드 6장

| # | 슬라이드 | 시간 | 핵심 |
|---|---|---|---|
| 1 | 표지 | 15초 | 프로젝트명·팀·한 줄 |
| 2 | 문제 | 45초 | "이런 일이 매장에서 일어난다" |
| 3 | 해결책 | 60초 | "우리는 이렇게 만들었다" |
| 4 | 데모 | 90초 | 실제로 돌려보기 (라이브!) |
| 5 | 효과 | 45초 | 시간·매출·만족도 정량/정성 |
| 6 | 다음 단계 | 45초 | 매장 적용 계획 + 한 줄 마무리 |

## 슬라이드별 가이드

### 1) 표지
```
[프로젝트명 — 큰 글씨]
"한 줄 부제"
팀명 / 팀원 4명 / 영업트랙
```

### 2) 문제 슬라이드
- **헤드라인**: "신혼 패키지 견적, 30분 걸립니다"
- 매장 실제 사진 또는 일러스트 1장
- 숫자 1개로 임팩트 (예: "주 5건 × 30분 = 매장당 2.5시간 손실")

### 3) 해결책 슬라이드
- 우리가 만든 것의 **구조도** 1장
- 에이전트 팀 다이어그램 (4개 박스)
- "어떤 사례를 조합했는가" 명시

### 4) 데모 슬라이드 (가장 중요)
- 화면 공유로 실제 실행
- 입력 → 출력 흐름 30초 안에 보여주기
- 데모 실패 대비 **백업 스크린샷** 준비

{: .important }
> 데모는 워크샵 전체에서 가장 임팩트 큰 순간입니다. 발표 전 **3번 리허설** 필수.

### 5) 효과 슬라이드
- **정량 효과**: 시간 단축·매출 증대 (실측 또는 추정)
- **정성 효과**: 신입도 사용 가능·실수 감소 등
- 비포/애프터 표 형식

### 6) 다음 단계
- **30일 챌린지** 계획 (Week 1·2·3·4)
- 본사·타 매장에 공유할 계획
- 마지막 한 줄: "우리는 ___을 ___으로 바꾼다"

## 발표자 큐시트

```
[0:00] 화면 켜기 + 자기 소개
[0:15] 문제 설명 (목소리 톤 ↑)
[1:00] 해결책 소개 (구조도 가리키며)
[2:00] 데모 시작! "지금 돌려볼게요"
[3:30] 효과 설명 (숫자에 집중)
[4:15] 마무리 (다음 단계 + 한 줄)
[4:50] 인사
[5:00] Q&A (선택, 1~2분)
```

## 발표 팁

- **데모 실패 대비**: 백업 스크린샷·동영상 미리 준비
- **목소리**: 평소보다 1.5배 크게
- **시선**: 청중 3명 정도 골라 번갈아 보기
- **마무리**: 박수 받을 만한 한 줄로 끝내기

## 평가 기준 (재확인)

| 항목 | 배점 | 핵심 질문 |
|---|---|---|
| 실용성 | 40 | 우리 매장에서 내일부터 쓰나? |
| 완성도 | 30 | 정말 작동하는가? |
| 창의성 | 20 | 새로운 시각이 있나? |
| 발표력 | 10 | 5분 안에 핵심 전달? |
