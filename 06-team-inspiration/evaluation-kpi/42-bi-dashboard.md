# 42. BI 대시보드 (bi-dashboard)
{: .fs-7 }

데이터 웨어하우스·KPI 정의·시각화·자동 보고를 통합 구성하는 BI 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 데이터 소스·웨어하우스, KPI 정의, 시각화, 자동 보고
- **출력 형태**: 대시보드 설계서, KPI 카탈로그, 자동 보고 양식
- **사용 빈도**: 신규 대시보드 구축·분기 갱신

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 데이터 엔지니어 | 소스·웨어하우스 매핑 |
| KPI 설계자 | KPI 정의·계산 공식 |
| 대시보드 빌더 | 시각화·레이아웃 |
| 보고서 자동화 | 정기 보고 양식·전송 |
| reviewer | KPI 정의·계산 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 데이터→KPI→대시보드→보고 흐름 |
| KPI 표준 방법론 | KPI 정의서·메타데이터 패턴 |
| 시각화 방법론 | 차트 선택·색상·접근성 기준 |

## 비민감 데이터 활용 시 고려할 점

- 실 운영 KPI 수치는 비율·집계 형태로 가공
- 개별 사원·고객 수준 데이터는 마스킹
- 미공시 지표는 권한 분리

## 검증 시 일반적으로 보는 항목

- KPI 정의·계산이 사내 표준과 일치하는가
- 대시보드 수치가 원천 데이터와 일치하는가
- 차트가 정보 위계를 정확히 전달하는가
