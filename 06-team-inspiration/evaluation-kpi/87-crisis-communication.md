# 87. 위기 소통 (crisis-communication)
{: .fs-7 }

상황 파악·메시지 전략·보도자료·Q&A·모니터링을 통합하는 위기 대응 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 상황 분석, 메시지 전략, 보도자료, 예상 Q&A, 미디어 모니터링
- **출력 형태**: 대응 메시지 세트, 보도자료, 미디어 모니터링 보고
- **사용 빈도**: 위기 발생 시·정기 시뮬레이션

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 상황 분석가 | 사실관계·이해관계자·영향 분석 |
| 메시지 전략가 | 핵심 메시지·톤·강도 설계 |
| 보도자료 작성자 | 공식 메시지 작성 |
| 미디어 모니터 | 보도·SNS 반응 추적 |
| reviewer | 사실·표현·법적 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 상황→메시지→보도→모니터링 흐름 |
| 메시지 톤 방법론 | 상황별 강·중·약 톤 |
| 모니터링 방법론 | 키워드·감성·확산 추적 |

## 비민감 데이터 활용 시 고려할 점

- 미공개 사실관계는 입력 금지 (법무 승인 후 활용)
- 피해자·당사자 식별 정보 보호
- 내부 의사결정 진행 사항은 권한 분리

## 검증 시 일반적으로 보는 항목

- 사실관계가 확인된 범위 안에서 표현되었는가
- 약속한 사항이 실제 실행 가능한가
- 법무·PR·경영진 승인 절차가 명시되어 있는가
