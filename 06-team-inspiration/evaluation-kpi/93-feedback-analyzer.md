# 93. 피드백 분석 (feedback-analyzer)
{: .fs-7 }

고객·직원 피드백을 감성·주제별로 분석해 인사이트를 도출하는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 피드백 수집, 감성 분석, 주제 분류, 트렌드·인사이트 도출
- **출력 형태**: 감성 분포 보고서, 주제별 인사이트, 액션 아이템
- **사용 빈도**: 분기 VOC·연간 직원 서베이 시점

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 데이터 수집가 | 채널별 피드백 통합·정제 |
| 감성 분석가 | 긍정·중립·부정 분류 |
| 주제 분류 전문가 | 핵심 토픽 추출 |
| 인사이트 작성자 | 우선순위 액션 아이템 |
| reviewer | 분류 정합성·표현 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 수집→감성→주제→인사이트 흐름 |
| 감성 분류 방법론 | 별점-텍스트 불일치 검출 |
| 인사이트 도출 방법론 | 빈도·강도·트렌드 종합 |

## 비민감 데이터 활용 시 고려할 점

- 응답자 식별 정보는 코드화·집계
- 강한 부정·법적 위험 피드백은 별도 처리
- 미공개 서비스에 대한 피드백은 권한 분리

## 검증 시 일반적으로 보는 항목

- 감성 분류 정확도(샘플 검수)가 기준 이상인가
- 주제 분류가 mutually exclusive한가
- 인사이트가 액션 가능한 수준으로 구체적인가
