# 32. 데이터 분석 프로젝트 (data-analysis)
{: .fs-7 }

탐색·정제·통계·시각화·보고서까지 데이터 분석 사이클을 다루는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 탐색적 분석(EDA), 데이터 정제, 통계 분석, 시각화, 보고서
- **출력 형태**: 분석 보고서, 시각화 자료, 데이터셋 메타정보
- **사용 빈도**: 분석 과제 건별

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 탐색 분석가 | EDA·기초 통계·이상치 식별 |
| 데이터 정제자 | 결측·이상치 처리·변환 |
| 통계 분석가 | 검정·회귀·예측 모델 |
| 시각화 전문가 | 차트·대시보드 구성 |
| reviewer | 분석 방법·결과 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 탐색→정제→통계→시각화→보고 흐름 |
| 통계 검정 방법론 | 가설 검정·신뢰구간·효과 크기 |
| 재현성 방법론 | 데이터·코드·결과 추적 |

## 비민감 데이터 활용 시 고려할 점

- 실 운영 데이터는 샘플링·익명화 후 활용
- 개인정보·민감 정보는 마스킹
- 미공개 모델·알고리즘은 권한 분리

## 검증 시 일반적으로 보는 항목

- 통계 가정이 데이터 특성에 맞는가
- 결론이 데이터 한계·표본 범위를 명시하는가
- 분석이 동일 데이터·코드로 재현 가능한가
