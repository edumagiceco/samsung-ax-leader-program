# 82. 업무 보고서 (report-generator)
{: .fs-7 }

데이터 수집·분석·시각화·집필·요약을 통합하는 정기 보고서 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 데이터 수집, 분석, 시각화, 본문 집필, 요약
- **출력 형태**: 주간·월간 보고서, 1장 요약 카드
- **사용 빈도**: 주간·월간·분기 정기 보고

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 데이터 수집가 | 출처별 데이터 통합 |
| 분석가 | 변화·이상치·인과 분석 |
| 시각화 전문가 | 차트·표 구성 |
| 보고서 작성자 | 본문·요약 집필 |
| reviewer | 수치·문맥 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 수집→분석→시각화→집필→요약 |
| 이상치 분석 방법론 | YoY·QoQ·MoM 변화 패턴 |
| 요약 작성 방법론 | 1장 요약·세부 부록 분리 |

## 비민감 데이터 활용 시 고려할 점

- 실 매출·KPI는 비율·등급 표기
- 개별 부서 수준 평가는 권한 분리
- 미공시 지표는 입력 금지

## 검증 시 일반적으로 보는 항목

- 본문 수치와 차트 수치가 일치하는가
- 변화 분석이 단순 비교 이상 인과를 짚는가
- 요약이 본문 핵심을 정확히 반영하는가
