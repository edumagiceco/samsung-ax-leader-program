# 48. 영업 지원 (sales-enablement)
{: .fs-7 }

고객 분석부터 제안서·프레젠테이션·팔로업까지 영업 사이클을 통합 지원하는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 고객 페르소나·니즈 분석, 제안서 초안, 프레젠테이션 스크립트, 사후 팔로업
- **출력 형태**: 견적서·제안서, 응대 스크립트, 팔로업 메시지
- **사용 빈도**: 고객 상담 건별·주간

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 고객 분석가 | 페르소나·니즈·예산·결정 기준 정리 |
| 제안서 작성자 | 제품 조합·견적 초안 작성 |
| 프레젠터 | 매장·법인 응대 스크립트 작성 |
| 팔로업 매니저 | 사후 메시지·일정 관리 |
| reviewer | 산출물 톤·정합성 교차 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 분석→제안→발표→팔로업 흐름 조율 |
| 견적 산출 방법론 | 패키지 할인·번들·할부 계산 기준 |
| 응대 화법 방법론 | 강·중·약 클로징 스크립트 패턴 |

## 비민감 데이터 활용 시 고려할 점

- 고객 실명·연락처는 코드화 (예: C001)
- 매출·할인율은 비율로 표현
- 미공개 가격 정책은 입력 금지

## 검증 시 일반적으로 보는 항목

- 견적 합계와 단가 단순 합이 일치하는가
- 응대 스크립트가 본사 가이드 톤과 맞는가
- 팔로업 일정이 영업 사이클 규정 안에 있는가
