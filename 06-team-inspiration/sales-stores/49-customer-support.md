# 49. 고객지원 시스템 (customer-support)
{: .fs-7 }

5/20 실습에서 사용한 도메인의 변형 — FAQ·응대 매뉴얼·에스컬레이션·분석을 통합하는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 5/20 실습 도메인과의 관계

- 5/20에 사용한 `customer-support` 하네스(시니어 고객 응대 매뉴얼 작성)와 같은 영역
- 이 카드는 harness-100 원본의 일반 구성을 안내. 5/20 실습용은 max-samsung-harness-samples 레포의 customer-support 폴더 참고

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 카테고리별 FAQ, 응대 매뉴얼, 에스컬레이션, CS 분석
- **출력 형태**: FAQ 리스트, 응대 매뉴얼, 에스컬레이션 흐름도, CS 인사이트 보고서
- **사용 빈도**: 분기 갱신·신제품 출시 시점

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| FAQ 빌더 | 카테고리별 자주 묻는 질문 정리 |
| 응대 전문가 | 상황·채널별 응대 스크립트 |
| 에스컬레이션 매니저 | 단계별 라우팅·SLA·권한 설계 |
| CS 분석가 | VOC·CSAT·NPS 지표 분석 |
| reviewer | 톤·정책 일관성 교차 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | FAQ→매뉴얼→에스컬레이션→분석 흐름 |
| CSAT 분석 방법론 | CSAT·NPS·CES 지표 설계 |
| 에스컬레이션 SLA 방법론 | 단계별 응답 시간·권한 정의 |

## 비민감 데이터 활용 시 고려할 점

- 고객 실명·연락처는 가명·코드 처리
- 컴플레인 내용은 핵심만 요약, 식별 정보 제거
- 미공개 보상 정책은 입력 금지

## 검증 시 일반적으로 보는 항목

- 응대 스크립트가 본사 가이드와 충돌이 없는가
- 에스컬레이션 흐름에 책임자·권한이 명확한가
- CS 지표 정의가 표준과 일치하는가
