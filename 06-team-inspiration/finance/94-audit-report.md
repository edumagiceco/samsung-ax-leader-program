# 94. 내부감사 보고서 (audit-report)
{: .fs-7 }

감사 범위 설정·체크리스트·발견 사항·개선 권고·추적을 통합하는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 감사 범위 정의, 체크리스트, 발견 사항, 개선 권고, 후속 추적
- **출력 형태**: 감사 보고서, 개선 추적 대장
- **사용 빈도**: 분기·연간 정기 감사

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 감사 범위 설계자 | 감사 영역·우선순위·일정 정의 |
| 체크리스트 작성자 | 영역별 점검 항목 작성 |
| 발견 사항 분석가 | 점검 결과·근본 원인 분석 |
| 권고 작성자 | 개선 권고·우선순위 |
| reviewer | 증빙·인용·표현 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 범위→체크리스트→발견→권고→추적 흐름 |
| 근본 원인 분석 방법론 | 5 Why·Fishbone 패턴 |
| 추적 대장 방법론 | 개선 항목 책임자·기한·상태 관리 |

## 비민감 데이터 활용 시 고려할 점

- 감사 대상 부서·인원 식별 정보는 코드화
- 미확정 발견 사항은 별도 분류
- 외부 공유 가능 영역과 내부 전용 영역 분리

## 검증 시 일반적으로 보는 항목

- 증빙 자료가 발견 사항과 정확히 연결되는가
- 근본 원인이 표면 원인과 구분되어 있는가
- 권고 사항이 측정 가능하고 추적 가능한가
