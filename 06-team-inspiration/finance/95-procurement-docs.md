# 95. 구매 문서세트 (procurement-docs)
{: .fs-7 }

요구사항 정의·벤더 비교·평가 기준·계약 조건·검수 기준을 통합 작성하는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 구매 요구사항, 벤더 비교표, 평가 기준, 계약 조건, 검수 기준
- **출력 형태**: RFP·계약서 초안, 평가 결과 보고서
- **사용 빈도**: 구매 건별

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 요구 정의자 | 기능·비기능·납기 요구사항 |
| 비교 분석가 | 벤더 제안 비교표 |
| 평가 기준 설계자 | 가중치·평가 척도 |
| 계약 검토자 | 표준 계약 조건·리스크 검토 |
| reviewer | 정책·규정 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 요구→비교→평가→계약→검수 흐름 |
| 평가 방법론 | 가중 점수·정성·정량 평가 |
| 계약 리스크 방법론 | 표준 조항·면책·종료 조건 |

## 비민감 데이터 활용 시 고려할 점

- 벤더 제안서·견적은 동의 범위 안에서 활용
- 가격·할인 정보는 비율 또는 등급으로 가공
- 미공개 입찰 정보는 입력 금지

## 검증 시 일반적으로 보는 항목

- 요구사항과 평가 기준이 일대일 매핑되는가
- 평가 점수 합계와 가중치가 정확히 계산되는가
- 계약 조건이 회사 표준·법규에 부합하는가
