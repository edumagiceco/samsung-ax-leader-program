# 64. 지식관리 체계 (knowledge-base-builder)
{: .fs-7 }

조직 지식관리 체계 구축을 위한 분류·위키·검색·유지보수 통합 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 지식 수집, 분류 체계, 마크다운 위키, 검색 인덱스
- **출력 형태**: 지식 베이스 구조도, 위키 페이지, 유지보수 가이드
- **사용 빈도**: 초기 구축 + 분기 갱신

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 지식 수집가 | 흩어진 문서·노하우 수집·정제 |
| 분류 설계자 | 도메인 분류 체계·태그 설계 |
| 위키 빌더 | 마크다운 위키 구조·페이지 작성 |
| 검색 최적화 전문가 | 검색 인덱스·키워드 매핑 |
| reviewer | 분류 일관성·중복 검출 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 수집→분류→위키→검색 흐름 조율 |
| 분류 체계 방법론 | 패싯 분류·태그 설계 기준 |
| 유지보수 방법론 | 갱신 주기·책임자·이력 관리 |

## 비민감 데이터 활용 시 고려할 점

- 사내 노하우·고객 사례는 익명화 후 수집
- 외부 공유 가능 영역과 사내 전용 영역 분리
- 영업 비밀·인사 정보는 별도 분류 등급 적용

## 검증 시 일반적으로 보는 항목

- 분류 체계가 mutually exclusive·collectively exhaustive인가
- 검색 시 의도한 문서가 상위 5개 안에 등장하는가
- 갱신 책임자와 주기가 명시되어 있는가
