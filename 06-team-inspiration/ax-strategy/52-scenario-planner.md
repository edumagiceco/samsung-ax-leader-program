# 52. 시나리오 기획 (scenario-planner)
{: .fs-7 }

핵심 변수 기반 시나리오 매트릭스를 만들어 영향 분석과 대응 전략을 도출하는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 핵심 변수 정의, 시나리오 매트릭스, 영향 분석, 대응 전략
- **출력 형태**: 시나리오별 의사결정 문서, 트리거 지표 정의서
- **사용 빈도**: 분기·반기·중대 변화 시점

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 변수 분석가 | 핵심 변수·불확실성 정의 |
| 시나리오 설계자 | 2×2 또는 다축 시나리오 매트릭스 설계 |
| 영향 평가자 | 시나리오별 영향·확률 평가 |
| 전략 수립자 | 시나리오별 대응 전략 작성 |
| reviewer | 시나리오 간 논리 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 변수→시나리오→영향→대응 흐름 조율 |
| 영향 평가 방법론 | 확률·영향·심각도 정량화 방식 |
| 트리거 지표 방법론 | 시나리오 전환점 감지 KPI 설계 |

## 비민감 데이터 활용 시 고려할 점

- 거시 환경(경기·정책) 데이터는 공개 출처 사용
- 내부 매출·인사 변수는 비율화 또는 익명화
- 경쟁사 시나리오는 공시 자료 기반

## 검증 시 일반적으로 보는 항목

- 시나리오들이 상호 배타적이고 망라적인가
- 각 시나리오의 전조 신호가 측정 가능한가
- 대응 전략이 실행 가능한 수준으로 구체적인가
