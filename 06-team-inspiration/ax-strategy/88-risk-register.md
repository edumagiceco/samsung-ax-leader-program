# 88. 리스크 관리대장 (risk-register)
{: .fs-7 }

프로젝트·조직의 리스크를 식별·평가·대응·모니터링하는 통합 관리대장 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 리스크 식별, 확률·영향 평가, 대응 전략, 모니터링 계획
- **출력 형태**: 리스크 관리대장, 상태 보고서
- **사용 빈도**: 월간·분기 갱신, 프로젝트 단위

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 리스크 식별자 | 카테고리별 리스크 발굴 |
| 평가 분석가 | 확률·영향 정량 평가 |
| 대응 전략가 | 회피·완화·전이·수용 전략 작성 |
| 보고서 작성자 | 관리대장·상태 보고서 작성 |
| reviewer | 리스크 분류 일관성·중복 점검 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 식별→평가→대응→보고 파이프라인 |
| 리스크 평가 방법론 | 확률·영향 매트릭스 기준 |
| 모니터링 KPI 방법론 | 리스크 트리거 지표 설계 |

## 비민감 데이터 활용 시 고려할 점

- 인사·보안 관련 리스크는 익명화·범주화 후 입력
- 재무 리스크는 비율·구간 형태로 가공
- 대외 협력사 명칭은 코드화

## 검증 시 일반적으로 보는 항목

- 모든 카테고리가 누락 없이 다뤄졌는가
- 평가 척도가 일관되게 적용되었는가
- 대응 전략이 실제 실행 책임자까지 명시되어 있는가
