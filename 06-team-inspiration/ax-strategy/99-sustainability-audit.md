# 99. ESG·지속가능성 감사 (sustainability-audit)
{: .fs-7 }

환경·사회·거버넌스 영역의 감사·보고서·개선 계획을 작성하는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 환경(E)·사회(S)·거버넌스(G) 각 영역 감사
- **출력 형태**: ESG 보고서, 개선 계획, 공시 자료
- **사용 빈도**: 반기·연간 보고

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 환경 분석가 | 탄소·에너지·자원 사용 감사 |
| 사회 영향 평가자 | 직원·고객·지역사회 영향 분석 |
| 거버넌스 검토자 | 의사결정·윤리·준법 체계 점검 |
| 보고서 작성자 | 통합 ESG 보고서 작성 |
| reviewer | 공시 기준 정합성 교차 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | E·S·G 3 영역 통합 흐름 조율 |
| ESG 공시 기준 방법론 | GRI·SASB·K-ESG 등 표준 매핑 |
| 개선 계획 방법론 | 우선순위·로드맵·KPI 설계 |

## 비민감 데이터 활용 시 고려할 점

- 임직원 개인 정보는 익명화·집계 형태
- 협력사·고객 데이터는 동의 범위 확인
- 미공시 ESG 지표는 입력 금지

## 검증 시 일반적으로 보는 항목

- 표준 공시 기준(GRI 등)과 매핑되는가
- 정량 지표와 정성 평가가 균형 있게 다뤄지는가
- 개선 계획이 차기 보고 시점까지 추적 가능한가
