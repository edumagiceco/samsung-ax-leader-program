# 26. IaC 설계 (infra-as-code)
{: .fs-7 }

Terraform·Pulumi 등 IaC로 환경 구성·보안·비용 최적화·드리프트 감지를 다루는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 클라우드 환경 구성, 보안 설정, 비용 최적화, 드리프트 감지
- **출력 형태**: IaC 코드, 환경 설계서, 비용·보안 리포트
- **사용 빈도**: 신규 환경 구축·분기 점검

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 인프라 설계자 | 네트워크·컴퓨팅·스토리지 구성 |
| 보안 엔지니어 | IAM·정책·암호화 설정 |
| 비용 최적화 전문가 | 인스턴스·예약·자원 효율 |
| 드리프트 감지자 | 실 환경과 IaC 정의 불일치 감지 |
| reviewer | 코드·보안·비용 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 설계→보안→비용→드리프트 흐름 |
| 보안 베이스라인 방법론 | CIS·NIST 가이드 매핑 |
| 비용 최적화 방법론 | 리저브드·스팟·라이트사이징 |

## 비민감 데이터 활용 시 고려할 점

- 실 운영 환경 자격 증명·비밀키는 절대 입력 금지
- 환경 설계는 사내 표준 안에서 안내
- 비용·예산은 비율 또는 등급 형태로 가공

## 검증 시 일반적으로 보는 항목

- IaC 코드가 사내 표준·보안 베이스라인을 따르는가
- 비용 추정이 실 사용량과 ±10% 범위에 있는가
- 드리프트 감지 시 알람·재배포 절차가 명확한가
