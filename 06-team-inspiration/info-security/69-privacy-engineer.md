# 69. 개인정보보호 (privacy-engineer)
{: .fs-7 }

GDPR·PIPA 분석·PIA·동의서·프로세스 설계를 통합하는 개인정보보호 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 개인정보보호 법규 분석, PIA(영향평가), 동의서 작성, 처리 프로세스
- **출력 형태**: PIA 보고서, 동의서 양식, 처리 흐름도, 개선 계획
- **사용 빈도**: 신규 서비스·시스템 도입·연간 점검 시점

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 법률 분석가 | GDPR·PIPA·관련 법규 매핑 |
| PIA 수행자 | 영향 평가·위험도 분석 |
| 동의서 작성자 | 동의서·고지문 작성 |
| 프로세스 설계자 | 수집·이용·파기 절차 |
| reviewer | 법적 정합성·표현 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 법규→PIA→동의서→프로세스 흐름 |
| PIA 평가 방법론 | 영향·발생 가능성·완화 평가 |
| 동의서 표현 방법론 | 명확성·구체성·이해 가능성 |

## 비민감 데이터 활용 시 고려할 점

- 실제 개인 식별 정보는 절대 입력 금지
- 가상·가명 사례로 시뮬레이션
- 미공개 신규 서비스 정보는 입력 금지

## 검증 시 일반적으로 보는 항목

- 동의서 표현이 법적 요건을 충족하는가
- PIA 결과의 완화 조치가 실제 적용 가능한가
- 처리 흐름이 수집·이용·파기 단계를 모두 다루는가
