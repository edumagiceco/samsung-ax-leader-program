# 28. 보안 감사 (security-audit)
{: .fs-7 }

취약점 스캔·코드 분석·침투 테스트·개선 권고를 통합 작성하는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 자산·취약점 스캔, 코드 분석, 침투 테스트, 보고서 작성
- **출력 형태**: 보안 감사 보고서, 개선 권고서, 우선순위 매트릭스
- **사용 빈도**: 분기·연간 정기 감사·인증 시점

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 취약점 스캐너 | 자산·서비스 스캔·CVE 매핑 |
| 코드 분석가 | 정적 분석·시큐어 코딩 점검 |
| 침투 테스터 | 시나리오 기반 모의 침투 |
| 보안 컨설턴트 | 우선순위·개선 권고 |
| reviewer | 위험도·증빙 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 스캔→분석→테스트→권고 흐름 |
| 위험도 평가 방법론 | CVSS·우선순위 매트릭스 |
| 개선 권고 방법론 | 단기·중기·장기 분류 |

## 비민감 데이터 활용 시 고려할 점

- 실 운영 시스템 정보는 절대 외부 입력 금지
- 취약점 상세는 사내 보안 등급에 따라 분리
- 감사 결과는 권한 있는 인원만 열람

## 검증 시 일반적으로 보는 항목

- 발견된 취약점에 재현 가능한 증빙이 있는가
- 위험도 평가가 공식 기준(CVSS 등)을 따르는가
- 개선 권고가 책임자·기한과 함께 명시되어 있는가
