# 67. 규정 준수 검증 (compliance-checker)
{: .fs-7 }

법률 매핑·현황 진단·갭 분석·개선 계획을 통합하는 컴플라이언스 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 법률·규정 매핑, 사내 현황 진단, 갭 분석, 개선 계획
- **출력 형태**: 컴플라이언스 점검 보고서, 갭 매트릭스, 개선 로드맵
- **사용 빈도**: 분기·연간 점검·규제 변경 시점

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 법률 분석가 | 적용 법률·규정 매핑 |
| 현황 진단자 | 사내 실태 점검 |
| 갭 분석가 | 요구 사항 대비 갭 식별 |
| 개선 계획 수립자 | 우선순위·로드맵 작성 |
| reviewer | 법률 해석·증빙 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 매핑→진단→갭→개선 흐름 |
| 법률 매핑 방법론 | 조문·고시·내규 연결 절차 |
| 우선순위 방법론 | 위험도·법적 의무·실행 가능성 |

## 비민감 데이터 활용 시 고려할 점

- 사내 정책·인사 데이터는 권한 분리
- 미공개 위반 가능성·진행 중 사안은 입력 금지
- 외부 공유는 법무 승인 후

## 검증 시 일반적으로 보는 항목

- 적용 법률 최신 버전이 반영되었는가
- 갭이 구체 조문·요구 사항과 매핑되는가
- 개선 계획이 실행 책임자·기한과 함께 정의되는가
