# 27. 데이터 파이프라인 (data-pipeline)
{: .fs-7 }

수집·변환·적재·품질 검증·모니터링을 통합 설계하는 데이터 파이프라인 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 데이터 수집(extract), 변환(transform), 적재(load), 품질 검증, 모니터링
- **출력 형태**: 파이프라인 설계서, 품질 체크 정의서, 모니터링 대시보드
- **사용 빈도**: 신규 데이터 소스 추가·분기 점검

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| ETL 설계자 | 소스·변환·타깃 흐름 설계 |
| 데이터 품질 관리자 | 품질 룰·이상치 검출 기준 |
| 스케줄러 | 작업 일정·의존 관계 |
| 모니터링 전문가 | 알람·지표·이상 감지 |
| reviewer | 데이터 흐름·스키마 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 수집→변환→적재→품질→모니터링 |
| 데이터 품질 방법론 | 완전성·정확성·일관성 룰 |
| 운영 모니터링 방법론 | SLA·알람 임계치 설계 |

## 비민감 데이터 활용 시 고려할 점

- 실 운영 데이터 스키마는 권한 분리
- 개인정보 포함 데이터는 마스킹·집계 후 활용
- 미공개 신규 데이터 자산은 입력 금지

## 검증 시 일반적으로 보는 항목

- 변환 로직이 소스·타깃 스키마와 정합한가
- 품질 룰이 실제 이상치를 잡아내는가
- 모니터링 알람이 false positive를 적절히 걸러내는가
