# 부서별 영감 라이브러리
{: .fs-8 }

**내가 매일 하는 일과 비슷한 사례**를 6 카테고리에서 찾아 본인 하네스의 출발점으로 삼습니다.
{: .fs-5 .fw-300 }

---

## 어떻게 쓰나?

1. 본인 부서가 속한 카테고리 클릭
2. 5개 사례 중 **내 일과 가장 비슷한 시나리오** 1개 선택
3. 그 카드를 본보기 삼아 [Harness-Lab](../04-harness-lab/) 시드 명령 작성
4. [AX 과제 브리프 워크북](../07-ax-brief-workbook/)에 적용 계획 기록

각 카드는 **5/20에 본 customer-support·real-estate-analyst와 같은 패턴**(5 agents + 3 skills)으로 통일되어 있어, 본인이 따라 만들기 쉽도록 설계되어 있습니다.

## 카드 표준 형식

각 카드는 다음 구조로 통일되어 있어, **본인이 매일 하는 일에 어떻게 변형할지** 한눈에 보입니다.

```
🎯 어떤 일에 쓰나요?
   - 매일/주간/월간 반복되는 구체 시나리오
   - 본인이 책상에 앉아 한 번에 처리하기 어려운 일

🤖 5명의 AI 직원 구성
   - 전문가 1: 데이터 수집·전처리
   - 전문가 2: 도메인 분석
   - 전문가 3: 위험·이상치 탐지
   - 전문가 4: 보고서/산출물 작성
   - reviewer: 교차 검증·일관성 점검

🛠️ 3개 스킬
   - 오케스트레이터: 5명을 어떤 순서로 일하게 할지
   - 확장 스킬 1: 도메인 방법론
   - 확장 스킬 2: 검증 방법론

📋 비민감 데이터 준비 가이드
   - 무엇을 익명화하고 무엇을 남길지

✅ 4주 검증 체크리스트
   - 본인 책상에서 매주 점검할 항목
```

---

## 카드를 본인 일로 변환하는 3단계

### 1단계 — 가장 비슷한 카드 찾기
- 본인 카테고리 페이지에서 5개를 빠르게 훑어보기
- 시나리오 한 줄이 **"아, 이게 내가 매일 하는 일이네"**라면 선택

### 2단계 — 5명의 전문가를 본인 일에 맞게 교체
- 카드의 5명 구성을 그대로 따르지 말고
- 본인 일에 필요한 **실제 4가지 전문성**을 적기
- reviewer는 그대로 유지 (5명 중 1명은 항상 검증 담당)

### 3단계 — Harness-Lab 시드 명령 만들기
- 위 4명 + 1 reviewer 구성을 1문장에 녹여
- [Harness-Lab](../04-harness-lab/)에서 실행
