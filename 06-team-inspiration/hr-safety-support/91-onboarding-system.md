# 91. 신규 입사자 온보딩 (onboarding-system)
{: .fs-7 }

체크리스트·교육·멘토 배정·30-60-90일 진행 추적을 다루는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 입사 첫 주 체크리스트, 직무 교육, 멘토 배정, 30/60/90일 진도
- **출력 형태**: 온보딩 가이드, 일자별 체크리스트, 진행 추적 대시보드
- **사용 빈도**: 입사 건별·분기 평가

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 온보딩 설계자 | 30/60/90일 계획 설계 |
| 교육 콘텐츠 빌더 | 직무별 학습 모듈 |
| 멘토 매칭자 | 신입-멘토 매칭 기준 |
| 진행 추적자 | 일자별 진도·달성도 점검 |
| reviewer | 부서 간 표준 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 설계→교육→멘토→추적 흐름 |
| 진도 점검 방법론 | 마일스톤·달성 기준 정의 |
| 멘토링 가이드 방법론 | 1:1 미팅 빈도·주제 패턴 |

## 비민감 데이터 활용 시 고려할 점

- 신입 식별 정보는 코드화
- 평가 결과는 인사 규정 범위 안에서 보관
- 부서 내 비공개 매뉴얼은 권한 분리

## 검증 시 일반적으로 보는 항목

- 30/60/90일 목표가 측정 가능한가
- 학습 모듈이 직무 활동과 연결되는가
- 멘토 매칭 기준이 객관적인가
