# 90. 채용 프로세스 (hiring-pipeline)
{: .fs-7 }

JD 작성·소싱·스크리닝·면접·평가·오퍼까지 채용 전 사이클을 다루는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: JD 작성, 소싱 채널, 서류 스크리닝, 면접 질문 설계, 평가 표준화
- **출력 형태**: JD 문서, 면접 가이드, 평가 매트릭스, 오퍼 메시지
- **사용 빈도**: 채용 공고 건별

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| JD 작성자 | 직무 기술서·자격 요건 정리 |
| 소싱 전문가 | 채널별 소싱 전략 |
| 스크리닝어 | 서류 기준·1차 필터 설계 |
| 면접 설계자 | 면접 질문·평가 항목 |
| reviewer | 차별 표현·공정성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | JD→소싱→스크리닝→면접→평가 흐름 |
| 면접 질문 방법론 | STAR·역량 기반 질문 패턴 |
| 평가 표준화 방법론 | 인터뷰어 간 점수 정합성 |

## 비민감 데이터 활용 시 고려할 점

- 지원자 이력·식별 정보는 코드화
- 평가 점수·이유는 인사 규정 범위 안에서 보관
- 차별·민감 정보(나이·성별·종교 등) 직접 활용 금지

## 검증 시 일반적으로 보는 항목

- JD 표현이 차별·편향 가이드를 위반하지 않는가
- 면접 질문이 직무 관련성을 가지는가
- 인터뷰어 간 점수 분포가 일관되는가
