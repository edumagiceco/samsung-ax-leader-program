# 84. 회의 전략 문서 (meeting-strategist)
{: .fs-7 }

안건·배경 자료·의사결정 프레임워크·회의록·팔로업까지 회의 전 사이클을 다루는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 안건 구조 설계, 배경 자료 조사, 의사결정 프레임워크, 회의록, 팔로업
- **출력 형태**: 안건서·진행 큐시트, 회의록 템플릿, 팔로업 액션 리스트
- **사용 빈도**: 주간·정기·중요 회의

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 안건 설계자 | 안건 구조·시간 배분 |
| 배경 조사원 | 관련 자료·이전 결정 사항 정리 |
| 프레임워크 설계자 | 의사결정 프레임워크 적용 |
| 문서 템플릿 빌더 | 회의록·팔로업 양식 |
| reviewer | 안건·회의록·팔로업 정합성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 안건→배경→프레임워크→회의록→팔로업 |
| 의사결정 프레임 방법론 | RACI·DACI·RAPID 패턴 |
| 액션 추적 방법론 | 후속 액션 책임자·기한 관리 |

## 비민감 데이터 활용 시 고려할 점

- 참석자 식별 정보는 직책·코드 표기
- 의사결정 미확정 사항은 별도 분류
- 인사·보안 안건은 권한 분리

## 검증 시 일반적으로 보는 항목

- 안건이 시간 안에 다룰 수 있는 분량인가
- 회의록에 결정·미결·액션이 구분되어 있는가
- 후속 액션의 책임자·기한이 명시되어 있는가
