# 83. 표준운영절차 (sop-writer)
{: .fs-7 }

프로세스 분석·절차서·체크리스트·교육 자료·버전 관리를 통합하는 SOP 작성 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 업무 프로세스 분석, 절차서 작성, 체크리스트, 교육 자료, 버전 관리
- **출력 형태**: SOP 문서, 체크리스트 카드, 교육 슬라이드
- **사용 빈도**: 신규 프로세스 도입·정기 갱신

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 프로세스 분석가 | 현재 프로세스 흐름·역할·시간 분석 |
| 절차서 작성자 | 단계별 절차·책임자·산출물 정리 |
| 체크리스트 설계자 | 실행·점검 카드 작성 |
| 교육 자료 제작자 | 교육용 슬라이드·시나리오 |
| reviewer | 절차·체크리스트·교육 일관성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 분석→절차→체크리스트→교육 흐름 |
| 절차서 표준 방법론 | 표제·번호·책임자 표기 패턴 |
| 버전 관리 방법론 | 개정 이력·승인·배포 절차 |

## 비민감 데이터 활용 시 고려할 점

- 사내 운영 노하우는 권한 등급 적용
- 외부 공개 가능 SOP와 내부 전용 분리
- 보안·안전 절차는 별도 검토 절차 유지

## 검증 시 일반적으로 보는 항목

- 절차 단계가 누락 없이 다뤄지는가
- 책임자·인수인계 지점이 명확한가
- 버전·승인 이력이 추적 가능한가
