# 61. 역량 모델링 (competency-modeler)
{: .fs-7 }

직무 분석·역량 사전·평가 루브릭·개발 계획을 통합 작성하는 하네스 사례.
{: .fs-5 .fw-300 }

---

## 어떤 도메인의 하네스인가?

- **다루는 영역**: 직무 분석, 역량 사전, 평가 루브릭, 개발 계획
- **출력 형태**: 역량 매트릭스, 평가 루브릭, 직무별 개발 로드맵
- **사용 빈도**: 연간 직무 체계 갱신·신규 직무 정의

## Agents 5명 구성 (예시)

| 에이전트 | 역할 |
|---|---|
| 직무 분석가 | 직무 수행 활동·필요 역량 도출 |
| 역량 사전 작성자 | 행동 지표·수준 정의 |
| 루브릭 설계자 | 평가 기준·점수 척도 |
| 개발 계획 수립자 | 역량 갭별 학습 경로 |
| reviewer | 직무·역량·평가 일관성 검증 |

## Skills 3개 구성 (예시)

| 스킬 | 역할 |
|---|---|
| 오케스트레이터 | 직무→역량→루브릭→개발 흐름 |
| 행동 지표 방법론 | BARS 등 행동 기반 척도 설계 |
| 개발 경로 방법론 | 70-20-10·학습 카탈로그 매핑 |

## 비민감 데이터 활용 시 고려할 점

- 인사 평가 결과는 익명화·집계 형태
- 미공개 조직 개편 계획은 입력 금지
- 임원·민감 직위는 별도 분류

## 검증 시 일반적으로 보는 항목

- 역량 정의가 직무 활동과 일대일 매핑되는가
- 평가 척도가 행동 지표로 측정 가능한가
- 개발 경로가 실제 학습 자원과 연결되는가
